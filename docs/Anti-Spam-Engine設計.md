# Anti-Spam Engine 設計

## 概要

低品質・汎用的な MD 投稿を D ランクとして非公開化し、マーケットプレイスの質を維持するエンジン。
D ランク判定時は改善アドバイスを自動生成し、クリエイターのレベルアップを促す。

## 実装ファイル

| ファイル | 役割 |
|----------|------|
| `src/lib/ai-auditor/index.ts` | D ランク判定ロジック追加 |
| `src/types/index.ts` | `Rank` に `"D"` 追加、`AuditResult.feedback` 追加 |

## D ランク判定条件

```typescript
const isDRank =
  !hasRunningCode          // 実行可能コードパターンが 3 種未満
  || ccaf.thoughtDensity < 30;  // 思考密度が低すぎる
```

※ `mdContent` が空文字の場合は D ランクを発動しない（後方互換性のため）

## フィードバックテンプレート

D ランク判定時に以下のテンプレートから自動生成：

1. 「コードが不足しています。`function`/`class`/`async` を 3 個以上含めてください」
2. 「具体性に欠けます。独自の工夫・閾値・エラー処理を 1〜2 段落で追記してください」
3. 「README の再録は別物です。実装ポイントの記述を増やしてください」

## ランク体系（更新後）

| ランク | 条件 | 公開 |
|--------|------|------|
| S | density≥70, uptime≥30d, intentSignals≥3, 実行コード ✓, テスト証跡 ✓ | ✓ |
| A | density≥60 または S 未満 | ✓ |
| B | baseline | ✓ |
| D | コード不足 または density<30 | ✗（非公開） |

## D ランクの visibility

D ランク MD は `/profile` で「未公開：D（下書き）」として表示。マーケットプレイスには掲載されない。
`audit()` の戻り値 `rank: "D"` を確認することで呼出し元がフィルタリング可能。
