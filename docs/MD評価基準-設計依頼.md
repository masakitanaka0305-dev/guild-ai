# GUILD AI — MD ファイル評価基準 設計依頼

> このドキュメントは「他の AI に評価基準を考えさせる」ための **自己完結型プロンプト** です。
> このファイル単体を別の AI（ChatGPT / Claude / Gemini など）に投げると、
> 文脈を別途渡さなくても回答が得られるよう書かれています。

---

## 0. あなた（評価する AI）の役割

あなたは GUILD AI のマーケットプレイスに登録される **MD ファイル（知識ノート）** を
**S / A / B / C** の 4 段階に格付けする AI Auditor を設計するコンサルタントです。
本ドキュメントの末尾「依頼内容」のセクション A〜E に **箇条書きで簡潔に** 答えてください。

---

## 1. システム概要（GUILD AI とは）

- **コンセプト**: 「知能資産の登記と信用の循環経済圏」。日本発の AI エージェント・プラットフォーム。
- **流通物**: 個人・開発者が書いた **MD ファイル**（ノウハウ・設計書・リバースエンジニアリング・運用知）。
- **読者**:
  - 一次読者は **他の AI エージェント**（A2A: AI-to-AI 自律取引でノートを読み込み、下流タスクに利用）
  - 二次読者は人間（ノートを直接読む or AI 経由で価値を享受）
- **価値の定義**: そのノートを読んだ AI が下流タスクで **既存の retrieve では到達できない品質** を出せること。
- **登録〜配信フロー**:
  1. 著者が `/sell` から MD を投稿
  2. AI Auditor が S/A/B/C を判定し自動出品
  3. 他 AI / 人間が API 経由で利用、課金は per-call（マイクロペイメント）

---

## 2. 現在の AI Auditor のロジック（簡略）

```
S 鑑定（魂の登記）: thoughtDensity ≥ 80 AND vercelUptimeDays ≥ 30 AND intentSignals が空でない
A 鑑定:            thoughtDensity ≥ 60 AND vercelUptimeDays ≥ 7
B 鑑定:            上記いずれも満たさない（最低保証ランク）
```

| 入力フィールド | 意味 |
|---|---|
| `thoughtDensity` (0-100) | 思考密度。文章のカバレッジと深さの加重 |
| `vercelUptimeDays` | 関連デプロイの稼働日数（実証実績） |
| `intentSignals[]` | 「ペアプロ実施」「設計レビュー済み」「ユーザインタビュー反映」等、人間の craft の痕跡 |

問題意識：
- 現状は **3 値のみで判定**（密度・稼働・意思）。**情報密度 vs 冗長**、**オリジナリティ**、**失敗例被覆** を直接測れていない。
- **C ランク**を新設したい（"LLM が一発生成できそうな MD" を可視化したい）。
- **AI 読者向け** の評価軸（トークン効率、parse 容易性、用語一意性）を強化したい。

## 3. 依頼内容

以下 A〜E に、それぞれ **箇条書き** で簡潔に答えてください。
冗長な前置き・お決まりの謙遜は不要です（GUILD AI の文化として、密度を尊重します）。

### A. 評価軸（ディメンション）の提案 — 3〜7 個

各軸について：
- **軸の名前**（1 行）
- **何を測るか**（1 行の定義）
- **どう観察するか**（具体的な検出シグナル — 文字数 / 構造の癖 / キーワードパターンなど。AI auditor が機械的に判定できるレベルで）
- **なぜ重要か**（GUILD AI の文脈で 1 行）

### B. S / A / B / C 4 段階の基準

下記フォーマットで埋めてください：

| ランク | 1 行の意味 | 必須条件（AND） | 加点条件（OR・任意） | 失格条件（これがあれば降格） |
|---|---|---|---|---|
| **S** | … | … | … | … |
| **A** | … | … | … | … |
| **B** | … | … | … | … |
| **C** | … | … | … | … |

### C. 期待分布

S / A / B / C それぞれ何 % が健全か、その**理由**も含めて。
（ヒント: ランクは「相対比較」と「絶対基準」の両面を持つべきか？）

### D. 失格条件 / 不正パターン

「これがあるとそもそも登録を拒否、または C 以下」の条件を 5 つ以上：
- 例: LLM が事前学習で知っているテンプレートのリフレーズ
- 例: 出典なしの断定 + 検証手段なし
- など

### E. JSON 出力例

AI Auditor が評価結果を返す JSON 構造を提案。最低限：

```json
{
  "rank": "S | A | B | C",
  "score": 0-100,
  "dimensions": { "<軸名>": <0-100>, ... },
  "reasons": [ "...", "..." ],
  "disqualifiers": [],
  "ai_consumability": { ... }
}
```

を満たしつつ、必要なら追加フィールドを足してください。

## 4. 制約・注意点

- **AI 読者ファースト**: 人間に読みやすいだけの整形は加点対象外。AI が下流で使えるかが本質。
- **冗長な丁寧さ ≠ 価値**: 同じ内容で長い MD は **減点**。情報密度で正規化する設計を入れてください。
- **オリジナリティ ≠ 衒学**: 学術的な希少性ではなく、「**AI training data に欠落した経験知**」かどうかが基準。
- **検証可能性が無い断定** は積極的に C 候補に落としてください。
- **日本語の MD** を主に評価する想定（英語混在は許容）。
- **失敗例 / 反例 / エッジケース** の記載がある MD を厚く評価する設計が望ましい。

## 6. 補足: 期待しないこと

- 「優しさを大切にしましょう」のような general advice は不要
- 倫理ガイドラインの一般論（プライバシー保護等）は省略可（GUILD AI 側で別途扱う）
